an assistant for pet care:
    * track pet care tasks (walks, feeding, etc)
    * consider constraints (time, priority, etc)
    * produce a daily plan and explanation


coding tasks: 
    1. design the UML system
    2. implement in Python
    3. connect it to Streamlit UI


core actions: 
    1. add a pet
    2. schedule a walk 
    3. see today's tasks


building blocks
    1. owner
        * contains subclass pet (can be many)
        * name 
        * schedule 
        * can perform task

    
    2. pet
        * animal type 
        * name 

    3. task 
        * type of task (walking, feeding, etc.)

    4. scheduler
        * list of tasks


design decisions:
    * v1: scheduler held the task list, each owner had their own scheduler
    * v2: moved task list to owner, made scheduler stateless
    * v3 (current): moved task list to pet, matching rubric requirement
        - tasks belong to the pet they're about (walk the dog -> belongs to the dog)
        - owner collects tasks across all pets via get_all_tasks()
        - scheduler takes an owner and retrieves tasks through it
    * also added to Task: due_time, completed status, mark_complete method (rubric required)
    * Pet is now a regular class (not dataclass) since it has behavior methods


implementation TODOs:

    * decide if explain_plan needs the original task list to compare against the plan


test plan:
    1. sorting correctness (happy path) - tasks added out of order come back in chronological order
    2. recurring daily task (happy path) - completing a daily task advances due date by 1 day, stays incomplete
    3. conflict detection overlap (happy path) - two tasks with overlapping durations produce a warning
    4. pet with no tasks (edge case) - generating a plan with an empty pet doesn't crash
    5. filter by pet (happy path) - only returns tasks for the named pet
    6. exact same time (edge case) - two tasks at identical start times are flagged as conflicts
